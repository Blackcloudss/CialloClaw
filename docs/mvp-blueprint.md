# CialloClaw MVP 设计蓝图（可直接进入设计与开发）

## 1. MVP 目标

构建“低打扰陪伴 + 高效率工具”的桌面端 AI 助手：

- 常驻桌面像素角色悬浮球
- 需要时快速进入聊天和快捷任务
- 在合适时机轻提醒
- 支持自动化任务与任务巡检
- 支持多 Agent 隔离（权限/记忆/路由）

## 2. 信息架构

- 悬浮球（根）
- 任务环（悬停展开，6-8 内）
- 聊天弹出层（左键）
- 右键系统菜单
- 控制面板（设置 + 任务巡检 + 安全边界）
- 全景操作仪表盘（实时事件流）

## 3. 状态机（像素角色映射）

- `idle`：常态/空闲，低频呼吸，低饱和
- `ready`：悬停就绪，边缘微亮
- `thinking`：思考中，表情专注 + 思考标记
- `executing`：执行中，稳定状态 + 进度感
- `attention`：提醒/需用户，短时强调
- `error`：错误/失败，红色告警
- `completed`：完成，短时庆祝后回归
- `paused`：暂停，弱动态
- `dnd`：低打扰，低透明低饱和
- `music`：长待机额外动作模式

## 4. 关键交互

### 4.1 悬浮球

- 左键：打开聊天弹出层
- 右键：打开系统菜单
- 悬停 250ms：展开任务环
- 离开 180ms：收起任务环
- 拖拽：释放后自动贴边

### 4.2 任务环

- 径向等角布局，方向记忆优先
- 每个任务球显示短标签和 tooltip
- 任务触发后写入聊天流程（可直接执行）

### 4.3 聊天弹出层

- 顶栏：状态 + Agent 切换
- 消息区：用户/助手/系统消息
- 输入区：Enter 发送，Shift+Enter 换行
- 快捷 Chips：总结/翻译/解释/待办/下一步

### 4.4 右键菜单

- 打开对话
- 暂停/恢复 Agent
- 打开控制面板
- 打开全景仪表盘
- 退出程序

## 5. 多 Agent 边界

- 每个 Agent 拥有独立：
- 记忆边界（会话历史隔离）
- 权限边界（绿/黄/红动作）
- 路由边界（生活/工作场景）

切换 Agent 时：

- 保留上一 Agent 历史
- 切换任务环快捷动作
- 重连 SSE（按 agent_id 过滤）

## 6. 主动提醒分级

- L0：仅状态变化
- L1：轻文本提醒（桌面）
- L2：长文本提醒（桌面）
- L3：富文本卡片（聊天内）
- L4：代操作确认流（必须确认）

MVP 已实现：L1/L2 提醒窗口 + 频率门控（最小间隔、每小时上限）。

## 7. 任务便签巡检

- 读取 `.md` 任务约定（Mock）
- 展示：状态、优先级、截止时间、建议下一步
- 支持启动时巡检与手动“立即巡检”

## 8. 全景操作仪表盘

实时事件面板，显示：

- 时间线
- 工具调用
- 证据/引用
- 推理摘要
- Browser 流
- Terminal 流
- File 流
- Decision 流

## 9. 事件协议（SSE）

服务端统一推送事件：

- `connected`
- `heartbeat`
- `agent.state`
- `run.started`
- `timeline`
- `tool.call`
- `citation`
- `decision`
- `reasoning.summary`
- `run.completed`
- `reminder.suggested`

## 10. 开发落地

### 后端（Go）

- HTTP API + SSE
- Mock Agent、Mock QuickTask、Mock NoteTask
- Chat 请求触发 Run 模拟流程
- 心跳和主动提醒后台任务

### 前端（WPF）

- MVVM（View + ViewModel + Service）
- SSE 客户端实时驱动 UI
- 不展示原生思维链，仅展示执行轨迹

## 11. 后续可扩展项

- 接入真实工具执行（browser/terminal/file）
- 引入真实权限审批流（黄/红动作）
- 接入本地任务文件监听（fs watcher）
- 真正的插件/Skills 安装与运行沙箱
- 托盘图标、系统通知中心、开机自启注册
